初学git的几点疑问

Nov 1, 2015

工作区 与 暂存区的区别 以及 git commit 与 git push的区别

工作区就是你电脑能看到的。

工作区里有个.git的隐藏目录。它就是所谓的版本库。几乎所有的git存储和操作的内容都位于其下。如果想备份库，注意要将其拷贝到其它地方。它里面就包含了stage(或叫index) files的暂存区。git会为我们创建第一个分支,叫master，以及有个指向master的指针Head

要强调一点。 git保存的不是文件差异或变化量 ，而是 一系列快照

git add 的时候会把文件修改提交到暂存区。

git commit 的时候才把其提交到master.这个只是保存到了本地的版本库

git push 的时候会把修改提交到远程版本库。

git rm 与 rm的区别

rm是物理删除掉工作区的文件。之后要执行git commit -am “msg”,其中a参数表示all,提交所有删除和修改的文件,但新增的不算.

执行完上面的操作就相当于执行了git rm.

那万一把工作区文件删错了,想恢复咋办.先执行git status,它会提示你Changes not staged for commit里deleted的文件有哪些,例如test.txt.然后你可以通过执行git checkout –test.txt,删掉如果是文件夹也可以有这个命令恢复.

在自己库再引入其它的库,怎么办

git clone 是不行的,提交时会报错.应该用git submodule add命令.

如何解决冲突

修改前没先git pull,导致了push时出现冲突.这时只要人工合并下冲突,再commit,再push就行.
